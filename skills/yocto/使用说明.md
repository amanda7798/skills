# Yocto Skill 使用指南

## 触发条件

当你的问题包含以下关键词时，会自动激活 Yocto skill：

### 核心关键词
- `yocto`, `bitbake`, `openembedded`, `poky`

### 文件类型
- `.bb`, `.bbappend`, `.bbclass`, `.conf`

### 配置文件
- `local.conf`, `bblayers.conf`, `layer.conf`

### 构建任务
- `do_compile`, `do_install`, `do_configure`, `do_fetch`, `do_patch`

### 工具命令
- `devtool`, `recipetool`, `bitbake-layers`

### 组件概念
- `meta-layer`, `recipe`, `BSP`, `distro`, `machine`, `image`

## 使用方式

### 显式调用
```
/yocto
```

### 自动触发示例

**Recipe 开发：**
```
如何创建一个 BitBake recipe？
我的 .bbappend 文件不生效
```

**构建错误：**
```
Yocto 编译失败，do_compile 报错
BitBake 提示 ERROR: Nothing PROVIDES 'xxx'
```

**配置问题：**
```
如何在 local.conf 中配置并行编译？
怎么添加新的 meta-layer 到 bblayers.conf？
```

**工具使用：**
```
devtool 如何修改现有 recipe？
recipetool create 的用法
```

## 提问建议

**包含这些信息会得到更好的帮助：**
- Yocto 版本（如 Kirkstone, Dunfell）
- 错误日志或具体报错信息
- 相关的 recipe 或配置文件内容
- 你的目标和当前状态

**示例：**
```
使用 Yocto Kirkstone，编译自定义应用时报错：
ERROR: Task do_compile failed
日志显示：error: xxx.h: No such file or directory

我的 recipe：
DEPENDS = "libxxx"
SRC_URI = "git://..."

目标：成功编译并打包到镜像中
```

## 功能范围

✅ 可以帮助：
- 编写和调试 recipe (.bb, .bbappend)
- 解决构建错误
- 配置 layer 和构建系统
- 使用 devtool/recipetool
- BSP 和内核定制
- 依赖关系问题

❌ 超出范围：
- 具体硬件调试
- 应用程序源码逻辑错误
- 非 Yocto 相关的嵌入式问题
